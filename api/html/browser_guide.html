Cypher: Introduction to NO-SQL

Cypher


Cypher is a graph-based query language for the Neo4J database.

Cypher can be used to write graph-based queries reminiscent of ASCII art.

A graph G is an ordered pair (V, E), where stem: V is the set of all vertices and E is the set of all edges.

Create a node


A node v of V can be represented by ().

This representation can be provided with the type. The operator : introduces such a type.

Here you see a node of the type person. Each person has a name. What name should the person have?:

Person 
CREATE (a:Person {name:'Rüdiger'})
RETURN a

Create a node of a different type


What name and attribute should the Pet have?:

Pet is a 
CREATE (a:Pet {name:'Rüdiger', type:'Dog'})
RETURN a

Create a between existing nodes


An edge can only exist between two node. An edge must always be directed. The type of edge can be determined within [].

is a friend of 
MATCH (a:Person{name:'Marc'}),(b:Person{name:'Rüdiger'})
MERGE (a)-[r:IS_FRIEND]->(b)
RETURN a,r,b

Create a between existing nodes of different type


An edge can only exist between two node. An edge must always be directed. The type of edge can be determined within [].

ownes the 
MATCH (a:Person{name:'Marc'}),(b:Pet{name:'Rüdiger', type:'Dog'})
MERGE (a)-[r:OWNES]->(b)
RETURN a,r,b

Selecting


An edge can only exist between two node. An edge must always be directed. The type of edge can be determined within [].

MATCH (a:Person)-[r:OWNES]->(b:Pet{type:'Dog'})
RETURN a,r,b
